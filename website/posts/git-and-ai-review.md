2026 年 5 月 13 日，我在日志里记录了对 Git 流程的初体验，也写下了一个观察：

「有个想法：AI 带来的强大生产力导致人力审核滞后于其提交内容量。就算引入了这套 Git 版本管理的流程，还是一样的问题。」

那天还写了：克隆、拉取、推送；从 main 拉功能分支、改完发 Merge Request；打包成动态库；模块封装。

## 我的困惑

工具升级了——Git、MR、动态库、分支策略——但**瓶颈从「写不出来」变成了「看不完、审不完」**。AI 可以很快产出一堆 diff，人有没有时间理解每一行？合并标准是什么？

我还没有标准答案。目前能做的是：小步提交、MR 里写清楚改了什么、能拆的模块尽量拆，让 review 的范围可控。

> 流程解决协作，不自动解决质量。生产力变强之后，「慢下来审一遍」反而更重要。
